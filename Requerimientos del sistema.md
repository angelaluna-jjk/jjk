FUNCIONALES
Iniciar Sesion: el sistema permite a el usuario registrar su nombre y una constraseña.
Seleccion de productos: el sistema permite a el usuario seleccionar el producto de su preferencia.
Carrito de compras: el sistema debe permitir al usurio agregar su producto al carrito, cantidad y eliminar.
Pago de productos: el sistema debe tener la capacidad de procesar los pagos mediante diferentes metodos.
Seguimiento de tiempo real: el sistema debe mostrar al usuario la ubicacion del repartidor en un mapa tras confirmar el producto.

NO FUNCIONALES
Rendimiento: el sistema debe cargar el historial de compras que ha hecho cada usuario y asi mismo tener la capacidad de recibir diferentes datos simultaneamente.
Disponibilidad: la aplicacion debe estar en funcionamiento las 24 horas del dia.
Seguridad: Todos los datos personales de los usuarios y registros de pago deben estar encriptados en una base de datos.
Usabilidad: la interfaz debe permitir al usuario interactuar con la pagina y realizar el pedido de su preferencia.
